## Purpose
Ensure end-to-end traceability from business needs to system delivery,
minimizing gaps, rework, and scope leakage.

## Traceability Concept
Every approved requirement must be:
- Designed
- Built or configured
- Tested
- Deployed
- Governed

## Traceability Mapping Model

| Phase | Artifact | Purpose |
|-----|--------|--------|
| Requirement | BR100 / BR150 | Capture and approve business needs |
| Process Design | BP080 / Process List | Align requirements with business flows |
| Design | MD050 / MD070 | Translate requirements into functional design |
| Build | CEMLI / Configuration | System realization |
| Test | TE050 / Test Cases | Validate requirement fulfillment |
| Deploy | CR / Handover | Controlled release to production |

## BA Responsibilities
- Maintain requirement lineage across phases
- Validate that no requirement is built without approval
- Support go-live decision with traceability evidence

## Managerial Value
- Clear audit trail
- Reduced delivery risk
- Transparent scope control
